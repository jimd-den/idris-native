## 3. QTT ENFORCEMENT PLAN

### Root Cause: Two Bugs in count_usage That Reject Valid Code

Bug 1 -- If branch counting sums instead of taking max:

  Current (wrong) in qtt_checker/mod.rs ~line 125:
    Term::BufferStore(b, i, v) | Term::If(b, i, v) => {
        self.count_usage(name, b) + self.count_usage(name, i) + self.count_usage(name, v)
    }

  This treats "if cond then x else x" as using x twice.
  Only one branch executes at runtime.
  Fix: max(count_usage(then_branch), count_usage(else_branch))

Bug 2 -- Recursive calls treat argument occurrences as independent usages:
  "ack m n = ack (m - 1) n" counts m as used twice because m appears
  in two positions in the same expression tree.
  Fix: track the function name being defined; recursive self-application
  arguments need a separate counting context.

Because of these bugs, ackermann.idr and sha256.idr FAIL QTT even though
they are valid Idris 2 programs. These must be fixed before mandatory
enforcement is meaningful.

---

### Design: Always-On QTT with Three Error Levels

Instead of a --qtt flag, QTT is always active but distinguishes three levels:

  [ERROR]   Hard multiplicity violation.
            Variable used 0x when it must be 1x, or 2x when it must be 1x.
            Compilation halts.

  [WARNING] Unrestricted variable in a context that could be linear.
            Annotation missing but inferable. Compilation continues.

  [NOTE]    Variable could be made linear (compiler optimization suggestion).
            Purely informational.

This matches how real QTT languages work: you do not opt into type safety,
you annotate to relax it.

---

### The Error Pipeline

Current state: the compiler panics or returns a bare bool on any error.
Target state: a structured error pipeline with Idris 2-style output.

  Source text
      |
      v
  [Scanner]      -> LexError    { line, col, char, message }
      |
      v
  [Parser]       -> ParseError  { line, col, token, expected, message }
      |
      v
  [QttChecker]   -> QttError    { variable, declared_mult, actual_uses, location, hint }
      |
      v
  [Diagnostics]  -> renders all errors Idris 2-style with carets + hints
      |
      v
  stdout

Each QttError contains:
  variable_name:        String
  declared_multiplicity: Multiplicity  (Zero | One | Many)
  actual_usage_count:   usize
  source_location:      Span { line, col, len }
  context:              String   e.g. "in function body of 'ack'"
  hint:                 String   e.g. "consider marking as unrestricted or restructure"

Example rendered output:

  Error: QTT multiplicity violation in 'ack'
    --> ackermann.idr:2:5
     |
   2 | ack m n = if m == 0 then n + 1 else ack (m - 1) 1
     |     ^
     | Variable 'm' declared linear (1) but counted as used 2 times.
     | Hint: In if/else branches, a linear variable may appear once in each
     |       branch -- fix count_usage to use max instead of sum.

---

### QTT Annotations in Source Syntax

For users writing explicitly QTT-annotated Idris 2:

  -- Explicit linear argument (used exactly once):
  ack : (1 m : Integer) -> (1 n : Integer) -> Integer

  -- Erased argument (compile-time only):
  id : (0 ty : Type) -> ty -> ty

  -- Unrestricted (default, no annotation):
  ack : Integer -> Integer -> Integer

The parser has a partial stub for (0 x : Type) and (1 x : Type) in parse_pi()
but it is not wired through to check_multiplicity.

---

### Source File QTT Compliance

Your existing .idr files do not use QTT annotations. With count_usage bugs fixed
they will pass automatically. For users writing annotated code:

  -- Compiles cleanly (used exactly once):
  linearUse : (1 x : Integer) -> Integer
  linearUse x = x + 1

  -- Fails with QttError (used twice):
  duplicate : (1 x : Integer) -> Integer
  duplicate x = x + x

qtt_fail.idr already has "duplicate x = x + x" -- the right test case.
It needs the error pipeline to report the violation clearly instead of
silently returning false.

---

### Implementation Priority Order

  1.  Fix count_usage for If branches             (unlocks ackermann + sha256 passing QTT)
  2.  Fix count_usage for recursive self-calls    (correct linear counting in recursion)
  3.  Structured error types -- no more panics    (makes the compiler usable)
  4.  Diagnostics rendering -- Idris 2 style      (makes errors developer-friendly)
  5.  Wire Pi multiplicity annotations through    (enables explicit QTT source syntax)
  6.  Implement elaborate_adt / interfaces        (needed for real Idris 2 programs)
  7.  Wire REPL to real parser + compiler         (after pipeline is stable)
  8.  Fix WASM / bare-metal stubs                 (after core pipeline is solid)
  9.  Implement missing LlvmBackend methods       (gen_print_ir, set_target, emit_to_file, etc.)
  10. Remove MVP comments as each is resolved     (track progress against this list)
